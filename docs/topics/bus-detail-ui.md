# 버스 상세 UI

> 이 문서는 **현재 상태**를 서술한다. 변경 이력은 [devlog.md](../devlog.md)를 참조한다.
> 마지막 갱신: 2026-07-12

## 진입과 주요 동작

- 홈·검색·즐겨찾기·노선도 정류장 시트의 버스 행을 탭하면
  `showUnifiedBusDetailModal()`이 통합 버스 상세 모달을 연다.
- 상세 모달 상단의 버스 번호 배지는 48dp 이상의 터치 영역을 가진 노선도 버튼이다.
  번호 오른쪽의 `Icons.route_rounded`가 이동 가능함을 표시하고, 툴팁과 TalkBack에는
  `N번 노선도 보기`로 노출된다. 배지 색은 노선 배지 색상 규칙을 따른다.
- 노선도 버튼은 현재 `routeId`와 `routeNo`로 `RouteMapScreen`을 열며, 상세 모달은
  교체되어 시스템 뒤로가기로 원래 화면에 복귀한다.
- 상세 모달의 즐겨찾기와 승차 알람 버튼은 각각 기존 저장소와
  `BoardingAlarmActions` 공용 흐름을 사용한다.

## 도착 정보 디자인 방향

### 해결하려는 문제

- `첫 번째 버스`와 `다음 버스`가 각각 별도 제목과 카드로 분리되면 같은 성격의
  도착 정보가 두 섹션처럼 보여 비교 흐름이 끊긴다.
- 순서를 나타내는 제목과 카드 내부 정보 사이의 여백이 커져 작은 화면에서 실제
  도착 시간과 현재 위치가 늦게 보인다.

### 현행 구조

- 도착 정보는 하나의 `도착 예정` 목록으로 묶고 우측에 현재 표시 수(`N대`)를 둔다.
- 각 카드 내부의 작은 순서 배지로 첫 항목은 `먼저 도착`, 이후 항목은
  `다음 도착`으로 표시한다. 사용자가 먼저 비교해야 하는 값은 도착 시간이며,
  현재 정류장과 남은 정류장은 그다음 위계로 배치한다.
- 도착 카드와 `먼저 도착`/`다음 도착` 라벨은 급행 여부와 무관하게 중립
  surface 계열을 사용한다. 급행 전용 빨간색은 도착 시간이 임박한 경우의 숫자
  표시(3분 이하)에만 남기고, 그 외 시간과 라벨은 중립으로 유지한다.
- 도착 카드와 내부 시간 타일의 모서리는 8dp 이하로 유지하며, 카드 오른쪽 승차
  알람 버튼은 시각 아이콘 크기와 무관하게 48×48dp 터치 영역을 보장한다.
- `저상` 표시는 보조 배지로 유지하고, 긴 정류장명은 한 줄 말줄임 처리해 도착
  시간과 알람 버튼의 고정 영역을 침범하지 않는다.

### 검증 기준

- 두 대가 있을 때 `도착 예정`, `2대`, `먼저 도착`, `다음 도착`이 한 화면 흐름에
  나타나고 과거 `첫 번째 버스`, `다음 버스` 섹션 제목은 나타나지 않는다.
- 도착 정보 갱신, 큰 글꼴, 긴 정류장명에서도 시간 타일과 알람 버튼 크기는
  변하지 않아야 한다.
